☸️ kubectl - Guia Prático

Introdução

O kubectl é a principal ferramenta de linha de comando para administração de clusters Kubernetes.

Com ele é possível:

  • Consultar recursos
  • Criar objetos
  • Aplicar configurações
  • Verificar logs
  • Executar comandos em containers
  • Realizar troubleshooting

Pré-requisitos

Verificar se o KUBECONFIG está carregado:

echo $KUBECONFIG

Verificar o contexto atual:

kubectl config current-context

Verificar conectividade com o cluster:

kubectl get nodes

Informações do Cluster

Listar nodes:

kubectl get nodes

Exibir mais detalhes:

kubectl get nodes -o wide

Listar namespaces:

kubectl get ns

Verificar versão:

kubectl version

Informações do cluster:

kubectl cluster-info

Trabalhando com Pods

Listar pods do namespace atual:

kubectl get pods

Listar pods de todos os namespaces:

kubectl get pods -A

Exibir mais informações:

kubectl get pods -o wide

Descrever um pod:

kubectl describe pod <pod>

Exemplo:

kubectl describe pod nginx-6c8b7d4d9b-j7l9h

Logs

Visualizar logs:

kubectl logs <pod>

Acompanhar logs em tempo real:

kubectl logs -f <pod>

Logs de um container específico:

kubectl logs <pod> -c <container>

Últimas 100 linhas:

kubectl logs --tail=100 <pod>

Acesso ao Container

Abrir shell:

kubectl exec -it <pod> -- bash

Caso o container não possua bash:

kubectl exec -it <pod> -- sh

Exemplo:

kubectl exec -it nginx-6c8b7d4d9b-j7l9h -- bash

Deployments

Listar deployments:

kubectl get deployments

Descrever deployment:

kubectl describe deployment <deployment>

Verificar rollout:

kubectl rollout status deployment/<deployment>

Reiniciar deployment:

kubectl rollout restart deployment/<deployment>

Exemplo:

kubectl rollout restart deployment/frontend

Services

Listar services:

kubectl get svc

Detalhar service:

kubectl describe svc <service>

Ingress

Listar ingress:

kubectl get ingress

Detalhar ingress:

kubectl describe ingress <ingress>

Aplicando Alterações

Aplicar manifesto:

kubectl apply -f deployment.yaml

Aplicar diretório inteiro:

kubectl apply -f manifests/

Remover recursos:

kubectl delete -f deployment.yaml

Troubleshooting

Verificar eventos

kubectl get events

Ordenar eventos por data:

kubectl get events --sort-by=.metadata.creationTimestamp

Pod não inicia

Verificar status:

kubectl get pods

Detalhar pod:

kubectl describe pod <pod>

Verificar logs:

kubectl logs <pod>

ImagePullBackOff

Possíveis causas:

  • Imagem inexistente
  • Imagem privada
  • Credenciais incorretas
  • Harbor indisponível

Verificar:

kubectl describe pod <pod>

CrashLoopBackOff

Possíveis causas:

  • Aplicação falhando na inicialização
  • Variáveis de ambiente incorretas
  • Banco de dados indisponível
  • ConfigMap ou Secret inválido

Verificar:

kubectl logs <pod>

Pending

Possíveis causas:

  • Falta de recursos
  • Node indisponível
  • Problemas de scheduling

Verificar:

kubectl describe pod <pod>
