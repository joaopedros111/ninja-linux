# 🌿 Git

## Introdução

Git é um sistema de controle de versão distribuído usado para registrar alterações em arquivos, colaborar em projetos e manter histórico de desenvolvimento.

No Ninja Linux, os conteúdos de Git ficam distribuídos entre esta área e a seção DevOps, pois Git também faz parte do fluxo de automação, versionamento e entrega contínua.

## Comandos de consulta rápida

```bash
git status
git add .
git commit -m "mensagem"
git log --oneline
git branch
git checkout nome-da-branch
git pull
git push
```

---

## Boas práticas

* Faça commits pequenos e descritivos.
* Verifique `git status` antes de adicionar ou commitar arquivos.
* Use branches para organizar mudanças.
* Evite commitar senhas, tokens e arquivos sensíveis.
* Atualize sua branch antes de abrir pull request.
* Revise o diff antes de enviar alterações.
